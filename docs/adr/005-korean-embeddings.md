# ADR-005: 한국어 특화 임베딩 모델 채택 (BGE-M3 / KURE-v1)

- **상태**: 채택 (Accepted)
- **날짜**: 2026-05-22
- **결정자**: 개발자
- **관련 요구사항**: F-1.3, 기술 스택

---

## 컨텍스트

RAG 검색 품질은 임베딩 모델 성능에 직결된다.
한국어 자소서 도메인에서 영어 중심 범용 모델 vs 한국어 특화 모델 중 선택이 필요했다.

---

## 결정

**BGE-M3 또는 KURE-v1(한국어 특화)을 임베딩 모델로 채택한다.**

| 모델 | 차원 | 특징 |
|------|------|------|
| BGE-M3 | 1024 | 다국어, 한국어 성능 우수, 오픈소스 |
| KURE-v1 | 1024 | 한국어 전용 특화, RAG 최적화 |

운영 환경에서는 BGE-M3 우선, 품질 비교 후 KURE-v1으로 교체 가능.

---

## 이유

한국어 자소서는 다음 특성이 있다:
- 직무 전문 용어 (예: "멀티에이전트", "LangGraph", "퍼셉트론")
- 한자어 + 영어 혼용 (예: "CNC 도메인에서 AI 솔루션 개발")
- 유의어 풍부 (예: "개발" = "구현" = "제작")
- 문장 맥락 중요 (단순 키워드보다 의미 유사도)

범용 영어 모델(nomic-embed-text, text-embedding-3-small)은 이 특성에서 성능이 낮다.

| 모델 | 한국어 BEIR 성능 | 비용 |
|------|----------------|------|
| BGE-M3 | 높음 | 무료 (로컬) |
| KURE-v1 | 매우 높음 (한국어 전용) | 무료 (로컬) |
| nomic-embed-text | 중간 | 무료 (로컬) |
| text-embedding-3-small | 중간 | $0.02/1M tokens |

---

## 대안 (검토 후 기각)

### 대안 1: nomic-embed-text (Ollama 기본)
- ❌ 기각: 한국어 성능 중간, 이미 영어 중심 학습

### 대안 2: OpenAI text-embedding-3-small
- ❌ 기각: API 비용 발생, 한국어가 영어 대비 성능 하락

### 대안 3: 한국어 BERT 계열 (KoSimCSE 등)
- ❌ 기각: RAG 용 임베딩 최적화 부족, 차원 불일치

---

## 결과

### 긍정적
- ✅ 한국어 자소서 RAG 검색 품질 향상
- ✅ 로컬 실행 → 비용 없음, 개인 데이터 외부 전송 없음
- ✅ pgvector Vector(1024)와 차원 일치

### 부정적
- ⚠️ 첫 실행 시 모델 다운로드 (~570MB for BGE-M3)
- ⚠️ CPU 임베딩 시 처리 시간 (GPU 있으면 해결)
