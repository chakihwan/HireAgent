# ADR-009: 공고 입력은 텍스트 붙여넣기 우선

- **상태**: 채택 (Accepted)
- **날짜**: 2026-05-22
- **결정자**: 개발자
- **관련 요구사항**: F-2.1, F-2.2

---

## 컨텍스트

채용 공고를 HireAgent로 가져오는 방법:
1. **텍스트 직접 붙여넣기**: 사용자가 사이트에서 복사 → 입력창에 붙여넣기
2. **URL 크롤링**: 공고 URL 입력 → 서버가 자동으로 내용 스크래핑

편의성은 URL 크롤링이 높으나, 여러 리스크가 있다.

---

## 결정

**공고 입력의 주(primary) 방식은 텍스트 직접 붙여넣기로 한다.**
**URL 크롤링은 보조(secondary) 기능으로, 공개 페이지 한정으로 지원한다.**

---

## 이유

| 리스크 | 텍스트 붙여넣기 | URL 크롤링 |
|--------|--------------|-----------|
| IP 밴 위험 | 없음 | 높음 (서버 IP 차단) |
| 사이트 정책 | 없음 | 이용약관 위반 가능 |
| 로그인 필요 공고 | 사용 가능 | 접근 불가 |
| UI 변경 영향 | 없음 | 스크래퍼 즉시 망가짐 |
| 구현 복잡도 | 낮음 | 높음 (사이트별 파서) |

국내 채용 사이트(원티드, 사람인, 잡플래닛, 점핏)는 모두 비공개 API 또는 크롤링 차단이 활성화되어 있다.

---

## 대안 (검토 후 기각)

### 대안 1: URL 크롤링 우선
- ❌ 기각: IP 밴 위험, 로그인 필요 공고 대응 불가, 파서 유지보수 비용

### 대안 2: 브라우저 확장프로그램으로 공고 추출
- ⚠️ Phase 2(F-8.6)에서 검토: 사용자 PC에서 실행이라 서버 IP 밴 없음

---

## 구현 방향

```
메인: 텍스트 붙여넣기 입력창 (큰 textarea, 사용자가 공고 텍스트 붙여넣기)
보조: URL 입력 → httpx로 공개 페이지 fetch → BeautifulSoup 파싱
      (로그인 없이 접근 가능한 URL만, 서버 단 실행)
```

---

## 결과

### 긍정적
- ✅ IP 밴, 크롤링 정책 위반 위험 없음
- ✅ 로그인 필요 공고도 붙여넣기로 처리 가능
- ✅ 어떤 채용 플랫폼이든 동작 (사이트 의존 없음)

### 부정적
- ⚠️ 사용자가 텍스트 복사-붙여넣기 동작 1회 필요

### 완화
- UX: 큰 입력창 + 포맷 가이드 제공
- Phase 2: 브라우저 확장프로그램으로 원클릭 전송
