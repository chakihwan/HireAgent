# ADR-002: 채용 사이트 자동 입력 미지원

- **상태**: 채택 (Accepted)
- **날짜**: 2026-05-22
- **결정자**: 개발자

---

## 컨텍스트

자소서 생성 후 채용 사이트(원티드, 사람인 등)에 직접 붙여넣기 하는 과정이 번거롭다.
브라우저 자동화(Selenium/Playwright)나 크롤러 스크립트로 자동 입력을 지원하는 방안을 검토했다.

---

## 결정

**채용 사이트 자동 입력을 지원하지 않는다. 사용자가 직접 복사-붙여넣기 한다.**

---

## 이유

| 리스크 | 설명 |
|--------|------|
| UI 변경 취약성 | 사이트 DOM 구조 변경 시 스크립트 즉시 망가짐 |
| IP/계정 밴 | 비정상 트래픽 감지 → 계정 차단 |
| 이용약관 위반 | 대부분 채용 사이트가 자동화를 명시적으로 금지 |
| 유지보수 비용 | 사이트별 셀렉터, 로그인 플로우 지속 관리 필요 |
| 책임 이슈 | 자동 제출 오류로 인한 사용자 피해 |

---

## 대안 (검토 후 기각)

### 대안 1: Selenium/Playwright 서버 사이드 자동화
- ❌ 기각: UI 변경에 취약, 서버 IP 밴 위험, 유지보수 비용 높음

### 대안 2: 브라우저 확장프로그램
- ⚠️ Phase 2(F-8.6)에서 재검토: 사용자 PC에서 실행이라 서버 IP 밴 위험 없음
- Phase 1 MVP 범위 외

---

## 결과

### 긍정적
- ✅ 사이트 UI 변경에 영향받지 않음
- ✅ IP 밴, 계정 차단, 법적 리스크 없음
- ✅ 개발 공수 절감 → 핵심 기능 집중

### 부정적
- ⚠️ 수동 복사-붙여넣기 필요

### 완화
- 항목별 "클립보드 복사" 버튼 제공 (M3 UI)
- Phase 2에서 브라우저 확장프로그램 재검토
