## 시나리오 1 — RAG 데이터 없이 자소서 생성 (베이스라인)

**목적**: RAG 없이 생성했을 때 품질 기준점 확인

1. `/projects` 페이지에서 **현재 인덱싱된 데이터 모두 확인**  
   - 검색창에 아무 키워드나 입력 → 어떤 데이터가 있는지 파악
2. `/generate` 페이지 이동
3. 실제 채용 공고 텍스트 붙여넣기 (사람인/잡코리아에서 복사)
4. 항목: `지원동기` / 글자수: `500` / 톤: `공식적` / 페르소나: `경력직`
5. 생성 → 결과 확인

**체크포인트**:
- [ ] progress 로그에 `[RAG N개 참고]`가 표시되는가? (0개면 RAG 미매칭)
- [ ] 본문에 `[참고 경험]`, `[경험 자료]`, `**볼드**` 같은 마크다운이 보이는가?
- [ ] 본문에 없는 회사명·수치·기술이 생성됐는가? (할루시네이션 확인)
- [ ] 평가 점수가 몇 점인가?

## 시나리오 2 — 이력서 업로드 후 RAG 인용 확인

**목적**: 본인 이력서를 인덱싱했을 때 자소서에 실제 경험이 반영되는지 확인

### Step 1 — 이력서 인덱싱

1. 이력서를 **MD 또는 TXT 파일**로 준비 (PDF도 가능, 텍스트 레이어 있는 것만)
2. `/projects` → **파일 업로드** 탭
3. 메타데이터 입력:
   - source_type: `이력서 (resume)`
   - category: `경력` 또는 `전체`
4. 파일 선택 → 업로드

### Step 2 — 검색 확인

검색창에 이력서에 있는 기술 키워드 입력 (예: `Python`, `FastAPI`, `데이터 분석`)  
→ 이력서 내용이 결과로 나오면 인덱싱 성공

### Step 3 — 자소서 생성

`/generate`에서 동일 공고로 다시 생성

**체크포인트**:
- [ ] progress에 `[RAG N개 참고]` N > 0이 표시되는가?
- [ ] 생성된 자소서에 이력서의 실제 경험(프로젝트명, 수치, 기술명)이 반영됐는가?
- [ ] 반영된 내용이 이력서에 없는 내용인가? (있으면 할루시네이션)
- [ ] 볼드/불릿/출처레이블 없이 순수 텍스트로 나오는가?

## 시나리오 3 — GitHub 레포 인덱싱 후 프로젝트 경험 인용

**목적**: GitHub README가 자소서 프로젝트 경험으로 자연스럽게 인용되는지 확인

### Step 1 — GitHub 레포 인덱싱

1. `/projects` → **GitHub Repo** 탭
2. 본인 GitHub 레포 URL 입력 (예: `https://github.com/chakihwan/HireAgent`)
3. 메타데이터:
   - source_type: `프로젝트 README (project_readme)`
   - project_name: 레포명
4. 인덱싱 실행

### Step 2 — 검색 확인

해당 레포의 핵심 기술/키워드로 검색 확인

### Step 3 — 백엔드 개발자 공고로 생성

```
[공고 예시 — 이 텍스트를 붙여넣기]
FastAPI 기반 백엔드 엔지니어를 모집합니다.
- Python, FastAPI 경험 필수
- PostgreSQL 또는 유사 DB 경험
- 멀티에이전트, LLM 연동 경험 우대
- Docker, CI/CD 경험 우대
```

**체크포인트**:
- [ ] README에 있는 수치/기술명이 에세이에 등장하는가?
- [ ] 등장한 내용이 README에 실제로 있는가? (없으면 할루시네이션)
- [ ] 자연스러운 문어체로 녹아들었는가? (복사 붙여넣기 아닌지)

## 시나리오 4 — 글자수 정확도 확인

**목적**: 압축/확장 에이전트가 목표 글자수 ±5% 내로 생성하는지 확인

1. 항목: `자기소개` / 글자수: `800`
2. 생성 후 결과 카드의 **글자수 배지** 확인
3. 항목: `지원동기` / 글자수: `300` (짧은 케이스)
4. 항목: `직무경험` / 글자수: `1000` (긴 케이스)

**허용 범위**:
- 800자 → 760~840자 OK
- 300자 → 285~315자 OK
- 1000자 → 950~1050자 OK

**체크포인트**:
- [ ] progress 로그에 compress 관련 메시지가 보이는가? (압축 시도 횟수 확인)
- [ ] 모든 케이스에서 ±5% 내로 생성됐는가?
- [ ] 압축 후에도 볼드/불릿이 없는가? (오늘 수정한 버그)

- 에러 토스트 대신 **SpaSiteGuide 카드** 표시
